# DevOps docs maintenance

Use this skill for operational checks, documentation, and project memory
maintenance that supports shipping.

## Workflow

1. Read existing docs, commands, CI config, and project memory.
2. Update only the docs or config relevant to the task.
3. Keep instructions command-oriented and factual.
4. Verify links, commands, and file paths when practical.
5. Record stable project knowledge in memory files.
6. Archive stale memory instead of deleting important history.

## Checks

- Setup instructions match the current project.
- Test/build commands are accurate.
- CI or deployment changes are documented.
- No secrets appear in docs or memory.
- Changelog or release notes are concise.

## Output

Return:

- Docs/config/memory changed
- Commands or checks run
- CI/deployment impact
- Remaining risks
